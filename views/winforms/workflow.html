Workflow to create a TradingApp or form-based analysis technique

Creating the UI using the Form Designer

The new Resource Editor makes it a snap to create analysis techniques that contain forms. There’s no need to manually enter lines and lines of code to add a control and set its properties. Simply double click on the control you want to add, and then sets its properties and events using the same Properties Editor used for other components

We’ll write the code for you.

To add a form to your analysis technique, you can simply select Add Form from the code editor’s shortcut menu.




Clicking on the ToolBox tabs will display a list of controls that can be added to the form.




Double clicking on a control will cause it to be added to the form.

Using the Visual Debugger

The Visual Debugger allows you to inspect the values of analysis technique variables, and monitor execution flow during run-time. You can easily set one or more breakpoints where you want your code to pause its execution, and then step through your code line by line, making it much easier to track down coding errors.

To get started, you simply set one or more breakpoints in your code using the Toggle Breakpoint menu item or pressing the F9 key from the new Debug menu.




After selecting your breakpoints, a red dot will appear next to each line where a breakpoint was set.




At this point, variables are available to be inspected using the new Autos window.
