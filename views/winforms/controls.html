Controls and the ContextMenu

Controls are user interface elements that are added to a WinForm and are used to allow users to perform various actions, as well as to present information to users. Controls can be manually created from their respective classes, and their properties and events individually coded, or the can be visually added to a WinForm using the TradeStation Development Environment’s Form Designer, where they can be easily resized and moved to their appropriate location. Following are the controls currently available to TradingApps.

Chart



The Chart control allows you to display one or more data series using a variety of 2D and 3D styles. The following chart types are available:

  • Area
  • Bar
  • BoxPlot
  • Bubble
  • CandleStick
  • Column
  • Doughnut
  • ErrorBar
  • FastLine
  • FastPoint
  • Funnel
  • Kagi
  • Line
  • Pie
  • Point
  • PointAndFigure
  • Polar
  • Pyramid
  • Radar
  • Range
  • RangeBar
  • RangeColumn
  • Renko
  • Spline
  • SplineArea
  • SplineAngle
  • StackedArea
  • StackedArea100
  • StackedBar
  • StackedBar100
  • StackedColumn
  • StackedColumn100
  • StepLine
  • Stock
  • ThreeLineBreak

To utilize the Chart control, you create it and add it to a form as with any other control. Additionally, you will need to create one or more ChartAreas to define the areas where plotting will take place. You will also need to create one or more ChartSeries, which defines the data to be displayed in the ChartArea. Optionally, axes and legends can be added.

TSChart

This control is based on the TradeStation Chart Analysis window, and offers a convenient way to display time series data.

CheckBox

Checkboxes allows users to select an option, and maintain a binary state of either on (checked) or off (unchecked).

ComboBox

Presents a list of items from which users can select. Clicking on the down arrow will display the list of items.

DataGridView

The DataGridView control allows you to display data in a tabular format.

Advanced functionality such as sorting is provided.

DateTimePicker

The DateTimePicker will display a calendar control allowing users to select a date and/or time. Clicking on the down arrow displays a calendar, from which a single or range of dates can be selected.

GroupBox

This control is provides a frame where related controls can be grouped. A name can optionally be supplied to further identify the controls in the group.

Label

A label control is used to display a non-editable text string in a form.

LinkLabel

This control displays a hyperlink, which a user can click to perform an action such as opening a web page.

ListView

ListView allows you to display a collection of items in a multi column format.

NumericUpDown

The NumericUpDown control represents an up-down control (also known as a spin box) that displays numeric values. The displayed value can be incremented or decremented by clicking on the up or down arrow.

Panel

Creates an area within a form where related controls can be grouped.

ProgressBar

The ProgressBar control offers visual feedback on the progress of a long running process. Once created and added to a form or other control, the Minimum and Maximum properties can be set and then the Value property, some value between the Minimum and Maximum, is updated at various points throughout the calculation. Once the calculation is complete, the control is typically hidden or destroyed.

Radio Button

The RadioButton control is typically grouped together with other RadioButtons to present a group of choices that are mutually exclusive. Unlike the CheckBox class, you may make only a single selection within a group of radio buttons.

Slider

With the Slider, users can easily select a value from a defined range by dragging a “thumb” along a horizontal or vertical bar. To use the Slider, simply create it, set Minimum and Maximum values, frequency of tick marks and an initial value. An event handler can also be supplied to allow an analysis technique to respond to changes in real-time.

TabControl

The TabControl provides a way to save space and avoid a cluttered and confusing UI when many controls, or controls belonging to different groups, are needed in a form.

TableLayoutPanel

The TableLayoutPanel control creates subsections within a form that will maintain the sizes in either absolute or percentage values as a form is resized, without requiring additional code to move and size the individual sections.

TextBox

The TextBox class is a form control that displays a text string that can be edited by the user. The text will wrap within the specified width and height of the control. Scroll bars will appear if the text wraps outside the specified width and height boundaries.

WebBrowser

The WebBrowser control allows you to duplicate IE functionality inside of a WinForm. This allows access to external web resources. Additionally, this control will allow you to display programmatically created HTML commentary, similar to the existing Analysis Commentary, but with much more flexibility and fewer restrictions. HTML-based TradingApps reside inside of this control.

ContextMenu

The context menu provides a convenient way to present a list of options or commands that can be selected by a user.
